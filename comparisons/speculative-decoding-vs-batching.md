# مقارنة: فك التشفير التكهني مقابل التجميع المستمر | Speculative Decoding vs Continuous Batching

## نظرة عامة | Overview

تقارن هذه الصفحة تقنيتين مختلفتين لتحسين كفاءة الاستدلال: فك التشفير التكهني (يركّز على تقليل الكمون) والتجميع المستمر (يركّز على زيادة الإنتاجية).

This page compares two different inference optimization techniques: Speculative Decoding (focuses on reducing latency) and Continuous Batching (focuses on maximizing throughput).

---

## 📊 جدول المقارنة التفصيلي | Detailed Comparison Table

| المعيار | فك التشفير التكهني | التجميع المستمر |
|---|---|---|
| **الهدف الرئيسي** | تقليل الكمون (Latency) | زيادة الإنتاجية (Throughput) |
| **المبدأ** | نموذج صغير يولّد مسودة، نموذج كبير يتحقق | إزالة الطلبات المنتهية وإدخال جديدة فوراً |
| **تحسين الكمون** | 2-3× تسريع | تحسين TTFT مع Prefix Caching |
| **تحسين الإنتاجية** | محدود | 23× (مقارنة بالتجميع الثابت) |
| **استخدام الذاكرة** | إضافي (نموذج مسودة) | أقل هدر (<4% مع PagedAttention) |
| **التعقيد** | متوسط (نموذجان متوافقان) | منخفض (مفعّل افتراضياً في vLLM) |
| **أفضل لـ** | تطبيقات تفاعلية (Chatbots) | خوادم API عالية الحمل |
| **متطلبات العتاد** | GPU واحد كافي (نموذجان) | GPU متعدد للاستفادة القصوى |
| **درجة الإثبات** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

## 🔍 تحليل مفصّل | Detailed Analysis

### فك التشفير التكهني (Speculative Decoding)
- **الميزة الأكبر:** تسريع 2-3× في توليد الرموز مع نفس المخرج بالضبط (Exact decoding).
- **العيب الرئيسي:** يتطلب نموذج مسودة (Draft Model) متوافق في مفرداته (Vocabulary) مع النموذج الرئيسي.
- **كيف يعمل:**
  1. النموذج الصغير (Draft) يولّد k رموز مسودة.
  2. النموذج الكبير (Target) يتحقق من جميع الرموز في خطوة واحدة متوازية.
  3. الرموز المقبولة تُعتمد، المرفوضة تُستبدل.
- **متى تستخدمه:** تطبيقات تفاعلية حيث الكمون هو الشاغل الرئيسي.

### التجميع المستمر (Continuous Batching)
- **الميزة الأكبر:** استغلال أمثل لوحدة GPU — لا وقت فراغ (Idle) بين الطلبات.
- **العيب الرئيسي:** يتطلب إدارة معقدة للذاكرة (يُدمج عادةً مع PagedAttention).
- **كيف يعمل:**
  1. عند كل خطوة تكرارية (Iteration)، يُفحص قائمة الطلبات.
  2. الطلبات المنتهية تُزال من الدفعة.
  3. طلبات جديدة من قائمة الانتظار تُضاف فوراً.
- **متى تستخدمه:** خوادم API تستقبل طلبات متعددة بأطوال مختلفة.

## 📋 التوصيات | Recommendations

| السيناريو | التقنية الموصى بها |
|---|---|
| Chatbot تفاعلي | فك التشفير التكهني + التجميع المستمر |
| API عالي الحمل | التجميع المستمر + PagedAttention |
| تطبيق RAG | التجميع المستمر + Prefix Caching |
| توليد دفعي (Batch) | التجميع المستمر (بدون Speculative) |
| موارد GPU محدودة | فك التشفير التكهني (نموذجان على نفس GPU) |

## ⚠️ ملاحظة هامة

التقنيتان ليستا متعارضتين! الأفضل هو استخدامهما معاً:
- التجميع المستمر يُحسّن استخدام GPU على مستوى الدفعة.
- فك التشفير التكهني يُسرّع توليد كل طلب فردي.
