# دليل تطبيقي: نشر استدلال بتكلفة أقل | Deploy Cheaper Inference

هذا الدليل مخصص لمهندسي الذكاء الاصطناعي الذين يرغبون في نشر نماذج لغوية كبيرة (LLMs) بتكلفة أقل مع الحفاظ على الأداء.

## 🎯 1. اختر محرك الاستدلال الصحيح

محرك الاستدلال يُحدّد كفاءة استخدام العتاد (Hardware). الخياران الرئيسيان حالياً:

### vLLM (مفتوح المصدر)
- يُستخدم PagedAttention لإدارة KV Cache بكفاءة <4% هدر.
- يُقدّم تحسين 2-4× في الإنتاجية مقارنة بـ Hugging Face Transformers.
- ✅ **الأفضل لـ:** أحمال عالية التزامن، نماذج عامة، بيئات متعددة العتاد.
- ❌ **ليس الأفضل لـ:** أحمال ذات إعادة استخدام عالية للسوابق (Prefix).

### SGLang (مفتوح المصدر)
- يُستخدم RadixAttention لإعادة استخدام KV Cache عبر الطلبات ذات السوابق المشتركة.
- يُقدّم تحسين يصل إلى 6.4× في الإنتاجية و 3.7× في الكمون لأحمال الوكلاء و RAG.
- ✅ **الأفضل لـ:** أحمال متعددة الدورات (Multi-turn)، وكلاء مستقلون (Agents)، RAG.
- ❌ **ليس الأفضل لـ:** أحمال ذات أوامر فريدة بدون سوابق مشتركة.

## 🎯 2. طبّق التجميع المستمر (Continuous Batching)

بدلاً من انتظار انتهاء كل الدفعة، أُدخل طلبات جديدة فور انتهاء الطلبات الحالية.

```bash
# vLLM (مفعّل افتراضياً)
python -m vllm.entrypoints.openai.api_server \
    --model meta-llama/Llama-3.1-8B-Instruct \
    --max-num-seqs 256

# SGLang (مفعّل افتراضياً)
python -m sglang.launch_server \
    --model-path meta-llama/Llama-3.1-8B-Instruct
```

**العائد المتوقع:** تحسين 23× في الإنتاجية مقارنة بالتجميع الثابت.

## 🎯 3. فعّل التخزين المؤقت للسوابق (Prefix Caching)

إذا كان حمل العمل يحتوي على سوابق مشتركة (مثل تعليمات النظام أو وثائق RAG):

```bash
# vLLM
python -m vllm.entrypoints.openai.api_server \
    --model meta-llama/Llama-3.1-8B-Instruct \
    --enable-prefix-caching \
    --gpu-memory-utilization 0.92
```

**العائد المتوقع:** 70-95% نسبة إصابة (Hit Rate) لأحمال RAG والوكلاء.

## 🎯 4. طبّق التكمية المناسبة

| السيناريو | الطريقة المُقترحة | التوفير |
|---|---|---|
| نماذج 70B+ على خوادم | SmoothQuant (W8A8) | ~50% ذاكرة |
| نماذج 7B-13B على أجهزة طرفية | AWQ (INT4) | ~75% ذاكرة |
| نماذج أي حجم مع الحفاظ على الدقة | LLM.int8() | ~50% ذاكرة |
| نماذج 7B-70B مع ضبط دقيق | QLoRA (NF4) | ~75% ذاكرة + تكلفة ضبط منخفضة |

## 🎯 5. استخدم فك التشفير التكهني (Speculative Decoding)

إذا كان الكمون (Latency) هو الشاغل الرئيسي:

```bash
# vLLM مع Speculative Decoding
python -m vllm.entrypoints.openai.api_server \
    --model meta-llama/Llama-3.1-70B-Instruct \
    --speculative_model meta-llama/Llama-3.1-8B-Instruct \
    --num_speculative_tokens 5
```

**العائد المتوقع:** 2-3× تسريع في توليد الرموز.

## 📋 ملخص الخطوات بترتيب الأولوية

1. **اختر المحرك** (vLLM أو SGLang حسب حمل العمل)
2. **فعّل التجميع المستمر** (مفعّل افتراضياً)
3. **فعّل Prefix Caching** (إذا كان هناك سوابق مشتركة)
4. **طبّق التكمية** (W8A8 أو INT4 حسب العتاد)
5. **أضف Speculative Decoding** (لتحسين الكمون)

## ⚠️ تحذيرات

- كل تقنية تقليل تكلفة لها تنازلات (Tradeoffs) — راجع كل إدخال في المكتبة لفهم المخاطر.
- لا تُطبّق كل التقنيات دفعة واحدة بدون قياس الأداء أولاً.
- اختبر على بيانات تمثيلية قبل النشر في الإنتاج.
