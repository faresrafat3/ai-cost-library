# قائمة الجودة | Quality Checklist

يُستخدم هذا الملف للتحقق من جودة كل إدخال قبل اعتماده.

This file is used to verify the quality of each entry before acceptance.

## ✅ للمدخلات التطبيقية | For Practical Entries

- [ ] **YAML Front Matter:** يحتوي على id, title_ar, title_en, type, status, category, subcategory, tree_path, cost_dimensions, proof_score, sources_count
- [ ] **الباب العربي:** مكتوب بعربية علمية واضحة، يحتوي على ملخص، أبعاد التكلفة، بوابات الأدلة، القيود، المصادر
- [ ] **الباب الإنجليزي:** محتوى أكاديمي متوافق مع الباب العربي (ليس ترجمة حرفية)
- [ ] **بوابات الأدلة الأربعة:** جميعها مُوثّقة (Built, Tested, Deployed, Saved)
- [ ] **درجة الإثبات:** مُصنّفة بوضوح (⭐⭐⭐⭐ / ⭐⭐⭐ / ⭐⭐ / ⭐)
- [ ] **أبعاد التكلفة:** مُحدّدة بدقة مع المصطلح العربي المقابل
- [ ] **المصادر:** Tier 1 أو Tier 2 على الأقل، بتنسيق صحيح
- [ ] **القيود والمخاطر:** مُذكَرة بوضوح وصدق
- [ ] **الأدوات:** مُدرجة الأدوات/المكتبات التي تطبّق الطريقة

## ✅ للمدخلات الناشئة | For Emerging Entries

- [ ] **YAML Front Matter:** كامل
- [ ] **الباب العربي والإنجليزي:** مكتوبان
- [ ] **حالة البحث:** مُوضّحة (معاينة، تجارب أولية، إلخ)
- [ ] **الأدلة الأولية:** ما يتوفر من نتائج
- [ ] **القيود:** لماذا لم تدخل مرحلة الإنتاج بعد؟
- [ ] **المصادر:** على الأقل مصدر Tier 2 أو 3
- [ ] **المسار المتوقع إلى الإنتاج:** متى وكيف يُتوقع أن تصبح تطبيقية؟

## ✅ للمدخلات النظرية | For Theoretical Entries

- [ ] **YAML Front Matter:** كامل
- [ ] **الباب العربي والإنجليزي:** مكتوبان
- [ ] **الأساس الرياضي/النظري:** مُوضّح
- [ ] **الفجوة بين النظرية والتطبيق:** مُحلّلة
- [ ] **المصادر:** على الأقل مصدر أكاديمي (Tier 2)
- [ ] **البحث المطلوب:** ما الذي يحتاج إثباتاً عملياً؟

## ✅ عامة | General

- [ ] لا يوجد نقل حرفي ضعيف في العربية
- [ ] كل مصطلح جديد مُضاف إلى GLOSSARY.md و data/glossary.json
- [ ] كل ادعاء رقمي مُرتبط بـ EVIDENCE_LEDGER.md و data/claims.json
- [ ] الملف يتبع سياسة عدم الترويج الزائف (NO_HYPE_POLICY.md)
- [ ] الروابط الداخلية بين الإدخالات صحيحة
