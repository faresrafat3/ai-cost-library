# مقارنة: LoRA مقابل QLoRA مقابل الضبط الدقيق الكامل | LoRA vs QLoRA vs Full Fine-tuning

## نظرة عامة | Overview

تقارن هذه الصفحة ثلاث طرق لضبط النماذج اللغوية الكبيرة على مهام محددة: الضبط الدقيق الكامل، LoRA، وQLoRA.

This page compares three approaches for adapting LLMs to specific tasks: Full Fine-tuning, LoRA, and QLoRA.

## 🔍 تحليل مفصّل | Detailed Analysis

### الضبط الدقيق الكامل (Full Fine-tuning)
- **الميزة الأكبر:** أعلى جودة ممكنة — كل المعاملات تتكيف مع المهمة.
- **العيب الرئيسي:** يتطلب موارد ضخمة (1120GB لـ 70B) — غير عملي لمعظم الفرق.
- **حجم نقطة التفتيش:** 14GB (7B) إلى 140GB (70B) لكل مهمة.
- **متى تستخدمه:** موارد غير محدودة + مهمة بالغة الحساسية.

### LoRA (Low-Rank Adaptation)
- **الميزة الأكبر:** تقليل المعاملات القابلة للتدريب 10,000× مع أداء مكافئ.
- **العيب الرئيسي:** لا يزال يحتاج تحميل النموذج الأساسي بدقة 16 بت (14GB للـ 7B).
- **حجم نقطة التفتيش:** ~35MB لكل مهمة (مقارنة بـ 350GB للضبط الكامل لـ GPT-3).
- **كمون الاستدلال:** صفر إضافي — يمكن دمج الأوزان في النموذج الأساسي.
- **متى تستخدمه:** معظم حالات الضبط الدقيق على موارد GPU متوسطة.

### QLoRA (Quantized LoRA)
- **الميزة الأكبر:** تمكين ضبط نموذج 65B على GPU واحد 48GB.
- **العيب الرئيسي:** أبطأ بنسبة 39% من LoRA بسبب أعباء فك التكمية.
- **حجم نقطة التفتيش:** نفس LoRA (~15-50MB).
- **متى تستخدمه:** موارد GPU محدودة جداً أو نماذج كبيرة جداً.

## 📋 توصيات حسب السيناريو | Scenario-Based Recommendations

| السيناريو | الخيار الأول | لماذا؟ |
|---|---|---|
| GPU واحد 24GB + نموذج 7B | LoRA | مناسب تماماً بدون تكمية |
| GPU واحد 48GB + نموذج 70B | QLoRA | الضبط الكامل يحتاج 1120GB |
| موارد غير محدودة + دقة قصوى | الضبط الكامل | كل المعاملات تتكيف |
| ضبط متعدد المهام (Multi-task) | LoRA | نقاط تفتيش صغيرة قابلة للتبديل |
| تدريب على بيانات قليلة (<1000) | LoRA + QLoRA | تقليل Overfitting |

## ⚠️ تحذيرات

- LoRA-FA يوفر ~27.8GB ذاكرة إضافية مقارنة بـ LoRA العادي أثناء ضبط LLaMA-7B.
- LoRA+ يحسّن جودة LoRA بتعديل معدل التعلم للمصفوفات A و B بشكل مختلف.
- لا تجمع QLoRA مع طرق تكمية أخرى — قد تتعارض.
